# NetMap — Session 2026-04-07

## Context

First session for netmap. Set up project structure and wrote the spec.

## Key Decisions

### Pure native, no frameworks

All paranoid mini-apps are pure native Android — Kotlin + Android Views (XML layouts). No Compose, no Hilt, no heavy DI frameworks. The goal is efficiency: low memory, small APK, fast startup.

NetMap requires Android-only APIs:
- `TelephonyManager` for cell info (RSRP, RSRQ, PCI, EARFCN)
- `FusedLocationProvider` for GPS tracking
- Foreground service for background recording
- Room for offline storage (lightweight enough to keep)
- MapLibre Native for map rendering

### Architecture

- Each mini-app is a separate Activity launched from the hub
- No DI framework — constructor injection, manual wiring
- Coroutines + Flow for async (built into Kotlin, zero overhead)
- `android.util.Log` instead of Timber
- `org.json` or kotlinx.serialization (no Gson/Moshi)

### What stays from original spec

- Data model (Recording, Measurement, CellMeasurement) — unchanged
- Room schema — unchanged
- TelephonyDataSource approach — unchanged
- LocationDataSource with FusedLocationProvider — unchanged
- Foreground RecordingService — unchanged
- Export formats (GeoJSON, CSV, KML, GPX) — unchanged

### What changed

- UI: Android Views + XML layouts instead of Compose
- DI: manual instead of Hilt
- No Accompanist (Compose-only library)
- No ViewModel (Activities manage state directly or use simple presenter)
- Tighter performance targets (< 50 MB idle, < 1s cold start)
