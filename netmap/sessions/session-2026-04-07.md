# NetMap — Session 2026-04-07

## Context

First session for netmap. Set up project structure and captured the full technical spec.

## Key Decisions

### Native vs WebView

NetMap requires Android-only APIs that cannot run in a WebView:
- `TelephonyManager` for cell info (RSRP, RSRQ, PCI, EARFCN)
- `FusedLocationProvider` for GPS tracking
- Foreground service for background recording
- Room database for offline storage
- MapLibre Native for map rendering

This makes it the first **native Compose screen** in paranoid, establishing a hybrid pattern:
- Simple apps → WebView (existing pattern)
- Native apps → Kotlin/Compose, launched via JS bridge from the hub

### Architecture Pattern

The spec prescribes MVVM + Clean Architecture with three layers:
1. **UI** — Jetpack Compose screens
2. **Domain** — use cases, models, repository interfaces
3. **Data** — Room, TelephonyDataSource, LocationDataSource, exporters

DI via Hilt. Async via Coroutines + Flow.

### Open Questions from Spec

1. Map provider: MapLibre (free, no API key) vs Google Maps (better tiles, billing)
2. Background recording: auto-resume after reboot?
3. Storage: auto-delete old recordings?
4. Privacy: encrypt recordings at rest?
5. Crowdsourcing: opt-in anonymous data sharing?
