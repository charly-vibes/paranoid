Architecture decision: paranoid will support two types of mini-apps: (1) WebView apps — vanilla HTML/CSS/JS, launched via WebView from the hub page (existing pattern), (2) Native apps — Kotlin/Compose screens requiring Android APIs, launched from the hub via a JavaScript bridge that calls into native code. The hub page (index.html) remains the entry point. When a user taps a native app, the WebView calls Android.launchNative('netmap'), which the MainActivity intercepts and navigates to the Compose screen. Back navigation returns to the WebView hub. This keeps the simplicity of web mini-apps while allowing native capabilities when needed.
